# KCML SSOT v1.4 Requirements Matrix

This matrix maps the SSOT v1.4 requirements, including the automatic onboarding
profile in chapter 37, to implementation and test evidence. Production release
remains blocked until the external infrastructure gates in the last row pass.

| Requirement area | Component | Database object | API/UI | Security measure | Automated evidence | Acceptance evidence |
| --- | --- | --- | --- | --- | --- | --- |
| Host isolation | Host router, MCP routes, nginx | `mcp_server.hostname` unique | `kcmlNNNN.hcasc.cz/mcp` | exact host lookup; default deny; preserved Host | `catalog.test.ts`, `onboarding-routes.test.ts` | unknown/admin/auth/register cross-host routes do not expose catalog data |
| Kaja/access token entropy | `security/secrets.ts` | `kaja_credential`, `access_token` | Kaja create, OAuth token | 64 CSPRNG bytes; purpose-separated HMAC | `secrets.test.ts` | Base64URL secret decodes to 64 bytes; plaintext is absent |
| Integration token lifecycle | `domain/onboarding.ts` | `integration_token` | token UI; admin token API | `kci_` + 512-bit entropy; one-time display; HMAC digest; fingerprint | `onboarding.test.ts`, `onboarding-db.test.ts` | initial 2h TTL, server heartbeat extension, 24h ceiling, revoke/delete/resume |
| One token / one job / one server | onboarding transaction | `integration_token`, `onboarding_job` | `POST /v1/onboardings` | unique current-token binding, idempotency, optimistic lock | `onboarding.test.ts`, `onboarding-db.test.ts` | concurrent duplicate POST returns the same job and cannot reserve a second identity |
| Programmer API isolation | `http/onboarding-routes.ts` | onboarding tables | `register.hcasc.cz/v1/onboardings` | generic 401; no-store; exact host; Bearer only | `onboarding-routes.test.ts` | missing/invalid token is 401 and route is absent on admin/unknown host |
| Strict manifest | `domain/registration.ts` | manifest JSON + digest | multipart `manifest` | strict schema 1.4, Node.js 22/TS, no unknown fields | `registration.test.ts` | unsupported runtime, retry, schema, limit and identity input are rejected |
| Safe source intake | `domain/upload-validation.ts` | `onboarding_source_revision` | multipart ZIP | quarantine-only extraction, digest/secret/dependency/path policy | `upload-validation.test.ts` | traversal, symlink, zip bomb, forbidden files/deps and secret patterns fail closed |
| PR and CI gates | GitHub App adapter, onboarding worker | `onboarding_gate`, job PR/check fields | job detail timeline | branch/path confinement, least-privilege PR CI, required checks | `onboarding-pr.yml`, `contract-test.mjs` | path-policy, schema, lint, typecheck, unit/contract, secret, SAST/SCA/license, SBOM and reproducible build |
| Signed supply chain | trusted main workflow, OCI verifier | job artifact/source/build fields | safe PR/CI/SBOM links | immutable digest, source commit/Actions run ID, cosign signature, subject-bound SBOM/provenance attestations | `oci.test.ts`, `onboarding-build.yml`, root CI build | mismatch or invalid signature/attestation leads to `QUARANTINED` |
| Isolated handler runtime | fixed supervisor image, per-call subprocess, OCI client | runtime/image fields on `mcp_server` | gateway-to-worker Unix socket | rootless, non-root, read-only, cap-drop, no-new-privileges, network none, CPU/RAM/PID/concurrency limits and hard subprocess timeout | `runtime-supervisor.test.ts`, fixed Dockerfile + supervisor/runner syntax/build checks | uploaded code never runs in web/supervisor process, cannot outlive timeout and receives no platform secrets |
| Controlled egress | egress proxy + per-job capability | `egress_capability` | `context.egress.fetch` | exact HTTPS host/port allowlist, DNS pinning, private/link-local/metadata block, no redirects | `egress-proxy.test.ts` | loopback, private, metadata, mapped IPv6 and unlisted hosts are rejected |
| Registration disabled | activation service | `mcp_server`, revision, statistics, monitoring profile | job state | transactional creation as disabled before public test | worker state-machine tests/typecheck | identity, hostname, resource, toolName and all digests are centrally assigned |
| OAuth/MCP public trial | activation service and MCP gateway | ephemeral Kaja/access tokens, events, logs | public HTTPS trial | permission checks, negative token cases, secret revocation | MCP route tests + staging release gate | initialize/initialized/tools/list/safe tools/call and negative token cases pass publicly |
| Logging and audit | MCP gateway, runtime log collector | `audit_event`, `runtime_log_event` | correlation timeline | structured fields, HMAC/fingerprint only, centralized redaction | `secrets.test.ts`, root CI | same correlation ID across gateway/auth/worker/audit and no integration/Kaja/access/egress secret |
| Monitoring and health | monitoring scheduler | `monitoring_profile`, `monitoring_probe_result`, statistics | dashboard probe timeline | DNS/TLS/readiness/metadata/synthetic probes plus inspection of the running Podman image digest and renewed signature verification | `oci.test.ts`, monitoring service typecheck + staging release gate | `HEALTHY` only after every required probe; actual runtime digest drift quarantines and stops the worker |
| Automatic activation | activation service + onboarding worker | job/server state and gate rows | job detail; server monitoring | only state machine may set `ACTIVE/enabled=true`; no bypass | onboarding state tests + staging release gate | complete PASS activates; failure disables; critical invariant failure quarantines |
| Home Assistant inventory | `home_assistant_device_inventory@1.0.0` | catalog-assigned `mcp_server` + identity-bound manifest + registration evidence | one-tool MCP resource, real OAuth→MCP UI test, lifecycle switch and exact response view | loopback-only upstream; HA token remains in agent; strict empty input; bounded output; per-server concurrency/rate limits; permission and server revocation | `home-assistant-device-inventory.test.ts`, `auth-permission.test.ts`, agent `test_device_inventory.py` | `docs/acceptance/home-assistant-device-inventory.md`, real MCP call and C-01…C-10/T-01…T-25 matrix |
| Admin UI | React dashboard | token/job/probe reads | Monitoring → automatic integration → token → job gates | session + CSRF, token one-time view, no token edit | UI typecheck/build + browser QA | desktop/mobile flow exposes fingerprint, TTL, job, PR/CI/deploy/test and final HTTPS resource |
| Database consistency | migration 005 and onboarding domain | all onboarding/runtime tables | all mutation APIs | FK/check/unique constraints, partial/FK indexes, short transactions, SKIP LOCKED leasing and continuous heartbeat | `onboarding-db.test.ts` in PostgreSQL CI | migration, concurrent idempotency, HMAC-only storage, leasing, rollback and cleanup |
| Deployment and HTTPS | systemd, preflight, nginx | migration table | register + KCML hosts | wildcard SAN check, exact/regex host blocks, default 444, worker/cosign/podman checks | `bash -n`, workflow validation, root CI | release is blocked without DNS-01 `*.hcasc.cz`, GitHub App, GHCR signing and rootless Podman |
